Definition

Conway-Wechsler system

By extending the standard -illion system for naming large numbers, million, billion, trillion, and so on, Conway and Guy published a system for naming N'th zillion numbers for N<1000. N-th zillion number is 103N+3 for short scale and 106N for long scale. See long and short scale for detail. Robert Munafo explains Conway-Wechsler system and examples in detail. Standard English name is used for N<10, and Conway wrote

You can find a name for any zillion from the 10th to the 999th by combining parts from the appropriate columns of the following table, and then replacing the final vowel by "illion":
N units tens hundreds
1 un N deci NX centi
2 duo MS viginti N ducenti
3 tre (*) NS triginta NS trecenti
4 quattuor NS quadraginta NS quadringenti
5 quin (**) NS quinquaginta NS quingenti
6 se (*) N sexaginta N sescenti
7 septe (*) N septuaginta N septingenti
8 octo MX octoginta MX octingenti
9 nove (*) nonaginta nongenti

*Note: when it is immediately before a component marked with S or N, "tre" increases to "tres" and "se" to "ses" or "sex" as appropriate. Similarly "septe" and "nove" increase to "septem" and "novem" or "septen" and "noven" immediately before components marked with M or N.

**quin is changed from original quinqua.

As Robert explains, the Conway-Wechsler system disagrees some dictionary name. Miakinen proposed that as quindecillion is a common name and Latin name of 15 is quindecim, not quinquadecim, quinqua should be replaced to quin, and a similar change to all the Conway-Wechsler names involving the quinqua- prefix should be updated. Robert adopted the suggestion, and call this modified version (quin) as the Conway-Wechsler system. I also follow this convention. Note that Miakinen also proposed to replace letters of e to é in some cases for French spelling, but it is irrelevant in English.

For N≥1000

Conway and Guy (1996) extended this system with Allan Wechsler for N≥1000 as follows.

With Allan Wechsler we propose to extend this system indefinitely by combining these according to the convention that "XilliYilliZillion" (say) denotes the (1000000X + 1000Y + Z)th zillion, using "nillion" for the zeroth "zillion" when this is needed as a placeholder. So for example the million-and-third zillion is a "millinillitrillion."

Modified system

Even after changing quinqua to quin, Conway-Wechsler names have some difference from the common names, for example, sedecillion and novendecillion are better known as sexdecillion and novemdecillion respectively. Many people simplified the Conway-Wechsler system, where Cookiefonster summarized as follows.

N units tens hundreds
1 un deci centi
2 duo viginti ducenti
3 tre triginta trecenti
4 quattuor quadraginta quadringenti
5 quin quinquaginta quingenti
6 sex sexaginta sescenti
7 septen septuaginta septingenti
8 octo octoginta octingenti
9 novem nonaginta nongenti

This way we don't need to memorize when to put a s, x, m, or n where, and it's faithful to the dictionary -illions.

Software

Self-made software for Python was used to produce the definition tables and the list of numbers in this page, and therefore they should be consistent. The software is published here under the MIT license. It can be used to obtain arbitrary N'th zillion number by using the extended system, as Python can handle arbitrary large integer.

How to use: Download zillion.py and make it executable. Then run the program with

zillion.py [-h] [-m] [-t] [-s] [-d] [N [N ...]]

where N means N'th zillion number. When 2 Ns are given, all values between them are shown. When more than 2 Ns are given, each values are shown. -m option uses the modified system. For other options, use -h option for help. For installation, just rename the file to zillion and copy the file to your system path.

It was confirmed that the output of zillion.py N for N<1000 exactly matches the list by Steve Olsen.
